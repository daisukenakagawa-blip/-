# 🎰 ジャグラー データ自動収集・分析ツール

東京都の**複数店舗**のジャグラーデータを毎日蓄積し、傾向を見える化して
店舗比較・翌日の狙い台候補をランキング表示するツールです（多店舗対応）。

> 「完璧な予想AI」ではなく、**データを正確に蓄積し、傾向を見える化する**ことを
> 目的としています。

---

## ⚠️ はじめに：利用規約・マナーについて（必読）

本ツールは公開データサイトからの取得を想定していますが、**実サイトへの接続は
初期状態では無効（デモデータ）**になっています。実サイトに接続する前に、必ず以下を
確認してください。

- 対象サイトの**利用規約**でデータの自動取得（スクレイピング）が禁止されていないか
- `robots.txt` で当該ページのクロールが許可されているか
- 公式にCSV等でデータ提供されている場合は、そちらの利用を優先する

安全のため、本ツールには次の仕組みを組み込んでいます。

- **1日1回まで**しか取得できないレート制限（`config.MIN_FETCH_INTERVAL_SEC`）
- `robots.txt` の自動チェック（`config.RESPECT_ROBOTS_TXT`）
- 連絡可能な `User-Agent` の付与
- 取得は**手動実行**が基本（自動化は任意）

最終的な順守責任は利用者にあります。規約が不明・禁止の場合は、デモデータのまま
分析機能をお試しください。

## セットアップ

```bash
cd jagler
pip install -r requirements.txt
```

## 使い方

### 1. Web画面（Streamlit）

```bash
streamlit run app.py
```

サイドバーの **「🧪 デモ履歴をまとめて生成」** を押すと、すぐに全機能を試せます。
画面は以下のタブ構成です。

| タブ | 内容 |
|------|------|
| 📋 データ一覧 | 取得データを台番号順に表示。**合算・REG確率を色分け** |
| 🏪 店舗傾向分析 | 台番号別傾向・前日凹み台・高REGランキング・過去30/90日平均 |
| 🔢 末尾分析 | 末尾別の平均合算・REG傾向 |
| 📅 曜日分析 | 曜日別の傾向 |
| 🎯 狙い台ランキング | 翌日の狙い台を1〜20位＋**おすすめ理由（日本語）** |
| 💾 CSVダウンロード | 全データ・ランキングをCSV出力（Excel対応） |

### 2. コマンドラインで取得（手動実行）

```bash
python collect.py                 # 本日分を取得
python collect.py 2026-06-09      # 指定日を取得
python collect.py --demo-history 90   # デモ履歴90日分を生成
```

## 取得・保存する項目

日付 / 曜日 / 台番号 / BIG回数 / REG回数 / 総回転数 / BIG確率 / REG確率 /
合算確率 / BB_REG合計 / REG比率 / 末尾

データは **`(日付, 台番号)` を一意キー**にして同日・同台の重複登録を防ぎます。
保存先は自動で切り替わります（CSV出力にも対応）。

| 保存先 | 用途 | 永続性 |
|--------|------|--------|
| **SQLite**（`data/jagler.db`） | ローカルPC | PC内に保存 |
| **Googleスプレッドシート** | クラウド公開時の蓄積先 | **再起動でも消えない** |

認証情報を設定するとスプレッドシートが「データベース本体」になり、
Streamlit Cloud で公開しても毎日の蓄積が消えません（設定は `DEPLOY.md` 参照）。

## 色分けの基準

**合算確率**（分母が小さいほど高設定示唆）
- 🟥 赤：1/109〜1/80
- 🟧 オレンジ：1/129〜1/110
- 🟨 黄色：1/140〜1/130

**REG確率**（マイジャグラーは REG が設定の鍵）
- 🟥 赤：1/255〜1/200 / 🟧 1/290〜1/256 / 🟨 1/330〜1/291

しきい値は `config.py` で自由に変更できます。

## 分析機能

- 曜日別傾向 / 末尾別傾向 / 台番号別傾向
- 前日凹み台分析（据え置き・翌日狙いの判断材料）
- 高REG台ランキング
- 過去30日平均 / 過去90日平均
- 翌日狙い台ランキング（1〜20位）＋ おすすめ理由

### 狙い台スコアの考え方（説明可能な加点方式）

```
スコア = 過去30日のREGの良さ      （マイジャグラーの肝）
       + 過去30日の合算の良さ
       + 末尾傾向
       + 翌日の曜日傾向
       + 前日凹み（据え置き期待）
```

ブラックボックスにせず、各台になぜ上位なのかを日本語の理由で表示します。

## 複数店舗（東京都の公開店）に対応する

巡回方式が2つあります（`config.CRAWL_MODE`）。

### Pattern A：1ページに全店データが並ぶサイト
`config.py` の `STORES` に店舗を手動で追加します。

```python
CRAWL_MODE = "A"
STORES = [
    {"name": "ビッグディッパー新橋1号店", "url": ".../{date}", "table_selector": "table.data"},
    # ... 公開している店を追加
]
```

### Pattern B：店舗一覧 → 各店ページを巡回（東京都の公開店を自動で辿る）
一覧ページから各店のリンクを自動で発見し、店ページを順に巡回します。
店舗を手で登録する必要はありません。

```python
CRAWL_MODE = "B"
AREA_INDEX_URL = "https://example.com/tokyo?page={page}"  # 東京都の店舗一覧
INDEX_PAGES = 5                       # 一覧が複数ページなら総ページ数
STORE_LINK_SELECTOR = "a.hall-link"   # 一覧内の各店リンク<a>のCSSセレクタ
STORE_BASE_URL = "https://example.com"  # 相対リンクの基準（任意）
STORE_PAGE_TABLE_SELECTOR = "table.data"  # 各店ページのデータ表
# 店ページに複数機種が混在する場合、機種名列を指定するとジャグラー系だけ残せる:
# COLUMN_MAP["machine_name"] = "機種"
# MACHINE_KEYWORDS = ["ジャグラー"]
```

どちらの方式でも、1リクエストごとに `REQUEST_DELAY_SEC` 秒の待機を入れ、全体で
1日1回に制限します（サーバ負荷軽減）。**多くの店が1つの集計サイトに集約されている**
ため連続アクセスになります。必ず規約・robots.txt の範囲で利用してください。

## 実サイト接続を有効にする（任意・要規約確認）

1. 対象サイトの規約・`robots.txt` を確認する
2. `config.py` を編集
   - `SCRAPER_ENABLED = True`（または環境変数 `JAG_SCRAPER_ENABLED=true`）
   - `STORES` に各店の `url`（`{date}` を含む）と `table_selector` を設定
   - `COLUMN_MAP`（列見出し → 内部カラムの対応）
3. サイト構造によっては `scraper.py` の `_parse_table()` を微調整

接続が許可されない（robots.txt 等）場合は自動的に取得を中止します。

## Googleスプレッドシート連携（任意）

1. Google Cloud でサービスアカウントを作成し JSON キーを取得
2. 対象スプレッドシートをサービスアカウントのメールアドレスと共有
3. `pip install gspread`
4. `config.py` の `GSHEET_ENABLED = True` と関連項目を設定
5. 画面サイドバーの「📊 スプレッドシート出力」から出力

## ファイル構成

```
jagler/
├── app.py          # Streamlit Web画面
├── collect.py      # CLI（手動実行のデータ取得）
├── config.py       # 設定（しきい値・接続先・色分け）
├── scraper.py      # 取得層（レート制限・robots.txt・解析）
├── storage.py      # 保存先ルーター（SQLite ⇄ スプレッドシート 自動切替）
├── database.py     # SQLite層（重複防止・派生指標計算）
├── gsheet_store.py # Googleスプレッドシートを蓄積先にする保存層
├── analysis.py     # 傾向分析・狙い台ランキング
├── exporter.py     # CSV / スプレッドシート出力・色分け
├── sample_data.py  # デモ用データ生成
└── requirements.txt
```
