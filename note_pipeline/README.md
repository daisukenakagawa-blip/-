# note 自動記事生産ライン（note_pipeline）

これまで作った4つのツールを**1本の生産ラインに連結**し、毎日自動で記事を生み出す体制です。

```
niches.csv（ニッチの待ち行列）
      │
      ▼
① note_factory  … リサーチ → 編集部AIが執筆 → 部長が精査（合格まで書き直し）
      ▼
② produced/ に集約 ＋ produced_log.csv に記録（重複生産を防止）
      ▼
③（任意）note_publisher … 下書き投稿
      ▼
   毎日自動実行（Windowsタスク）
```

## 設計思想（“ゴミ”を作らないための前提）

- **飽和テーマは入れない**。`niches.csv` には、`note_niche` が「狙い目」と出した、
  競合の弱い具体ニッチだけを並べます。土俵選びで9割決まります。
- **既定は「下書きまで」で止める**（`PUBLISH_MODE="none"`）。理由は明確で、
  **売れる最後の一押し＝あなたの実体験1行**は人間が入れるべきだから。
  完成記事には「（ここにあなたの実体験）」の空欄が残ります。そこを埋めて投稿します。
- 全自動公開も可能ですが（`--publish publish`）、実体験が入らないぶん刺さりにくくなります。

## 使い方

### 事前準備（最初に1回）
1. `note_factory` を動くようにする（`note_factory/①セットアップ.bat` ＋ `.env` に APIキー）
2. （投稿まで自動にするなら）`note_publisher` を準備（`①セットアップ.bat`＋`②ログイン.bat`）

### 毎日の生産
| ファイル | やること |
|---|---|
| `①記事を生産.bat` | niches.csv から本日分（既定1本）を生産し produced を開く |
| `②毎日自動生産を設定.bat` | 毎朝9時に自動生産するようWindowsタスクに登録 |
| `③毎日自動生産を解除.bat` | 上記の解除 |
| `候補を探す.bat` | note_niche を回して狙い目ニッチの候補を出す（キュー補充用） |

### コマンド
```bash
python pipeline.py                  # DAILY_COUNT 本を生産
python pipeline.py --count 3        # 本数指定
python pipeline.py --dry-run        # 実行計画だけ表示（API呼び出しなし）
python pipeline.py --publish draft  # この実行だけ下書き投稿まで
python pipeline.py --discover       # note_niche で候補を探索
```

## 運用サイクル（おすすめ）

1. **週1回**：`候補を探す.bat`（または note_niche）で狙い目ニッチを発掘 → 良いものを
   `niches.csv` に転記して**キューを補充**。
2. **毎日**：`②毎日自動生産を設定.bat` で自動生産。`produced/` に記事が貯まる。
3. **公開前**：各記事の「（ここにあなたの実体験）」を1〜2行埋める → note で有料設定 → 公開。
   （`note_publisher` で下書き投稿まで自動化しておくと、note上で開いて仕上げるだけ）

## 設定（`config.py`）

| 項目 | 説明 | 既定 |
|---|---|---|
| `DAILY_COUNT` | 1回（=1日）の生産本数 | 1 |
| `PUBLISH_MODE` | none / draft / publish | `none` |
| `INTERVAL_BETWEEN` | 連続生産の間隔（秒） | 5 |
| `PYTHON` | 実行コマンド（環境により `python3`） | `python` |

## キュー（`niches.csv`）

| 列 | 意味 |
|---|---|
| `theme` | 記事テーマ（タイトルの素） |
| `price` | 推奨価格（空欄ならAIが提案） |
| `genre` | リサーチ/検索に使う語（空欄なら theme） |

初期値として、競合が弱めの具体ニッチを数件入れてあります。
**あなたが少しでも経験/知識のある分野**に差し替えるほど、売れる記事になります。

## コストと注意

- 1本あたり `note_factory` の4役（スカウト/リサーチ/ライター/部長）＋書き直しが走るため、
  **Anthropic APIの従量料金**がかかります。まず `DAILY_COUNT=1` で様子を見てください。
- 各ツールの公開エンドポイント（note検索）やブラウザ自動操作は、**常識的な頻度**で。
  note の利用規約・各種法令（景表法・薬機法等）を守ってください。
- 自動生産は強力ですが、**最終的な品質と公開判断は人間（あなた）**が持ちます。

## ファイル構成

```
note_pipeline/
├─ pipeline.py        オーケストレーター（4ツールを連結）
├─ config.py          本数・投稿モード・各ツールの場所
├─ niches.csv         作るニッチの待ち行列
├─ produced/          完成記事の集約先
├─ produced_log.csv   生産済み記録（重複防止）
└─ ①〜③.bat           生産・毎日自動実行の登録/解除
```
