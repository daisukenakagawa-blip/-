# USD/JPY 独自自動売買ボット (MT4/MT5不要)

MetaTraderプラットフォームに依存せず、**Pythonコードだけで完結する**ドル円自動売買システムです。
戦略ロジック・バックテスト・実発注のすべてを自前のコードで持ちます。

- **バックテスト** … 追加ライブラリ不要・APIキー不要。`python3 run_backtest.py --synth` で即実行できます。
- **実トレード** … OANDA(v20 API)に接続して発注します。MT4/MT5アプリは一切不要です。

戦略は MT5版 `USDJPY_TrendPullback_MT5.mq5` と**完全に同一**です:

| 項目 | 内容 |
|---|---|
| 環境認識 | 4時間足 EMA50 / EMA200 (50>200=買い環境 / 50<200=売り環境) |
| 押し目判定 | 1時間足 RSI14 (買い:35〜45 / 売り:55〜65) |
| エントリー | 15分足の直近高値/安値ブレイク |
| 損切り | ATR14 × 1.5 |
| 利確 | ATR14 × 3.0 (最低RR 1:2、下回れば見送り) |
| リスク | 1トレードあたり口座資金の1% (数量を自動計算) |
| 停止条件 | 重要指標前後60分 / 日次 −3% / 月次 −10% |
| 禁止事項 | ナンピン・マーチンゲール (常に最大1ポジション・固定リスク) |

```

fx_bot/
  config.py            全パラメーター
```

依存ライブラリはありません(Python 3.8+ の標準ライブラリのみ)。

## 1. バックテスト

### すぐ試す (合成データ)

```bash
python3 run_backtest.py --synth
```

実データが無くても、内蔵の合成データ(トレンドとレンジが交互に来る乱歩)で
エンジンの動作を確認できます。※合成データはランダムなので成績に意味はありません。
**本当の検証は必ず実データで**行ってください。

### 実データで検証

15分足のヒストリカルデータをCSVで用意します(列: `time,open,high,low,close`)。
`time` は ISO8601(`2025-01-02T03:15:00Z`)か unix秒。

```bash
python3 run_backtest.py --csv usdjpy_m15.csv --trades
```

データの入手例:
- OANDAのAPIから取得(`fx_bot/oanda.py` の `candles()` を利用)
- Dukascopy / HistData などの無料ヒストリカルデータ

### 主なオプション

```bash
python3 run_backtest.py --csv data.csv \
  --balance 1000000 \   # 初期資金(円)
  --risk 1.0 \          # 1トレードのリスク%
  --spread 0.8 \        # スプレッド(pips)
  --breakout 20 \       # M15ブレイク参照本数
  --trades              # 個別トレード明細を表示
```

出力例:

```
==================== バックテスト結果 ====================
初期資金       : 1,000,000 円
最終資金       : ...
純損益         : ...
総トレード数   : ...
勝率           : ...
プロフィットF  : ...  (1.2以上が目安)
最大ドローダウン: ...%
==========================================================
```

**運用前のチェック**: プロフィットファクター1.2以上、最大ドローダウンが許容範囲内、
かつトレード数が十分(最低でも数十件)あることを、複数の期間
(直近1年/2年、円安局面、円高局面)で確認してください。

## 2. 実トレード (OANDA)

### 準備

1. OANDAで口座を開設し、v20のAPIトークンを発行
2. 環境変数を設定:

```bash
export OANDA_TOKEN=あなたのトークン
export OANDA_ACCOUNT=101-xxx-xxxxxxx-xxx
export OANDA_ENV=practice      # practice=デモ / live=本番
```

### 動作確認 (発注しない)

```bash
python3 run_live.py --once     # 今の相場で1回だけ判定を表示
python3 run_live.py --dry      # 常時稼働するが発注はしない
```

### 稼働

```bash
python3 run_live.py            # 新しい15分足が確定するたびに判定し、条件成立で発注
```

24時間動かす場合は、VPSやクラウド上で `nohup python3 run_live.py &` や
systemd / tmux などで常駐させてください。自分のPCで動かす場合は、
PCを起動し続ける必要があります(プログラムが止まれば新規エントリーも止まります。
ただしSL/TPはOANDAサーバー側に置かれるので、決済自体は実行されます)。

## 3. スマホで状況を見る (モバイルダッシュボード)

スマホのブラウザで開いて、残高・ポジション・現在のシグナル判定・損益を
確認できる監視画面です(発注はしません。発注は `run_live.py` が担当)。

```bash
python3 dashboard.py            # http://localhost:8000 で起動
```

`OANDA_TOKEN` / `OANDA_ACCOUNT` を設定しておくと、ライブの残高・ポジション・
現在のH4トレンド/H1 RSI/シグナルが表示されます。30秒ごとに自動更新します。

### スマホから開けるURLにする方法

ダッシュボードは「サーバーで動かして、公開URLをスマホで開く」形になります。
こちらから固定のリンクを発行することはできない(あなたのOANDA口座と
サーバーが必要なため)ので、URLは下記いずれかの方法で**あなたが取得**します。

**A. まず手軽に試す — ngrok (自分のPCで起動して一時URLを作る)**
```bash
# 1) PCでダッシュボードを起動
python3 dashboard.py
# 2) 別ターミナルで ngrok を起動 (https://ngrok.com で無料登録)
ngrok http 8000
```
→ 表示される `https://xxxx.ngrok-free.app` をスマホのブラウザで開くだけ。
PCを起動している間だけ有効です。

**B. 常時動かす — クラウドにデプロイ (24時間の本命)**
Railway / Render / Replit などにこのリポジトリをデプロイすると、
`https://あなたのアプリ名.up.railway.app` のような公開URLがもらえます。
- 起動コマンド: `python3 dashboard.py`(ポートは環境変数 `PORT` を自動使用)
- 環境変数に `OANDA_TOKEN` `OANDA_ACCOUNT` `OANDA_ENV` を設定
- ボット本体も動かすなら、同じ環境で `python3 run_live.py` も常駐させる

**C. VPSを持っている場合**
VPS上で `python3 dashboard.py` を常駐(systemd/tmux/nohup)させ、
`http://サーバーIP:8000` をスマホで開く。

### ★公開URLには必ずパスワードを
残高が表示されるため、URLを公開する場合は環境変数でトークンを設定してください。
```bash
export DASH_TOKEN=好きな長い文字列
python3 dashboard.py
```
スマホでは `https://....app/?token=好きな長い文字列` で開きます。

### スマホのホーム画面に追加すれば「アプリ風」に
スマホのブラウザでURLを開き、共有メニューから「ホーム画面に追加」すると、
アイコンをタップするだけで開けるようになります(専用アプリ不要)。

## 重要な注意点

1. **「安定した成績」を保証するものではありません。** どんな戦略にも負ける相場があります。
   必ずデモ(practice)で十分な期間検証してから、失っても困らない資金で始めてください。
2. **損益計算はJPY建て口座前提**です(OANDA Japan口座はJPY建て)。
   USDJPYの損益はそのまま円で計算しています。USD建て口座を使う場合は
   `backtest.py` の損益計算とロット計算の調整が必要です。
3. **重要指標フィルターはバックテストでは無効**です。OANDA v20には経済カレンダー
   APIが無いため、`oanda.py` の `has_high_impact_news()` は常にFalseを返します。
   実運用で厳密に使うには、外部の経済指標APIを接続してこのメソッドを差し替えてください。
4. このコードは教育・検証目的のものです。実資金での利用は自己責任で行ってください。
