# note 売れる記事リサーチツール

note で **実際に売れている / 人気のある記事** を自動で集めて分析し、
「どんなテーマ・タイトル・価格が売れているか」をレポートにまとめるツールです。

これは **note 自動記事量産ツールの第1ステップ**(リサーチ)です。
ここで得た売れ筋データ(CSV)を、次のステップ(記事の自動生成)の入力に使えます。

## できること

- `keywords.csv` に書いたキーワード/ジャンルで note を横断検索
- **有料記事(=実際に売れている記事)** と人気記事を収集
- 以下を自動でレポート化(Markdown):
  - 売れている有料記事ランキング(スキ数順)
  - 人気記事ランキング(タイトルの参考用)
  - **価格帯の分布**(いくらで売れているか / 中央値)
  - **タイトルに頻出する売れるワード**
  - タイトルの型(数字入り率・【】等の装飾率・平均文字数)
  - (任意)AI による「売れる理由・狙い目テーマ・推奨価格」の考察
- 収集した全記事を **CSV** にも保存(次の記事生成ツール用)

## Windows の方へ(ダブルクリックだけ)

| ファイル | やること |
|---|---|
| `①セットアップ.bat` | Python の部品を自動インストール(最初に1回) |
| `②リサーチ実行.bat` | リサーチを実行し、結果フォルダを自動で開く |

> 「WindowsによってPCが保護されました」と出たら「詳細情報」→「実行」を押してください。
> Python 未インストールの場合は https://www.python.org/ から入れてください
> (インストール時「Add Python to PATH」に必ずチェック)。

## 使い方(コマンド)

```bash
pip install -r requirements.txt
python research.py                  # keywords.csv のキーワードでリサーチ
python research.py 副業 NISA AI活用   # キーワードを直接指定することも可能
```

実行すると `output/` に次の2ファイルが作られます:

- `note_research_YYYYMMDD_HHMMSS.md` … 読みやすいレポート
- `note_research_YYYYMMDD_HHMMSS.csv` … 全記事の生データ(Excel で開けます)

## リサーチするキーワードの変え方

`keywords.csv` を編集して、調べたいジャンルを1行に1つ書くだけです
(`#` で始まる行はコメント)。スマホからでも GitHub 上で直接編集できます。

```
副業
ブログ 収益化
NISA
AI 活用
```

## 設定のカスタマイズ(`config.py`)

| 項目 | 説明 | 初期値 |
|---|---|---|
| `MAX_ITEMS_PER_KEYWORD` | 1キーワードあたりの取得件数 | 100 |
| `INCLUDE_FREE_NOTES` | 無料記事も分析に含めるか | True |
| `MIN_LIKE_COUNT` | これ未満のスキ数の記事は除外 | 10 |
| `TOP_N` | ランキングの表示件数 | 30 |
| `REQUEST_INTERVAL` | リクエスト間隔(秒) | 1.0 |
| `USE_AI_INSIGHT` | AI 考察を付けるか | True |

## AI 考察(任意)

`.env.example` を `.env` にコピーし、`ANTHROPIC_API_KEY` を設定すると、
AI が売れる理由・狙い目テーマ・推奨価格を要約します。
**設定しなくても**、ルールベースの分析レポートは出力されます。

```
ANTHROPIC_API_KEY=sk-ant-...
```

## 仕組み・注意

- note の **公開検索エンドポイント**(ブラウザの検索と同じ)を、人の閲覧と同程度の
  ゆるやかな間隔で利用してメタデータを集めています。`REQUEST_INTERVAL` で間隔を
  調整できます。短時間に大量アクセスしないようご注意ください。
- 取得できるのはタイトル・価格・スキ数などの**公開メタデータ**で、有料記事の
  本文は取得しません。
- note 側の仕様変更で取得できなくなる場合があります。その際は時間をおいて再実行
  するか、`modules/note_client.py` のエンドポイント/フィールド名をご確認ください。

## ファイル構成

```
note_research/
├─ research.py            メイン(これを実行)
├─ config.py              設定
├─ keywords.csv           調べるキーワード一覧
├─ requirements.txt
├─ .env.example           AIキー設定の雛形(任意)
├─ modules/
│  ├─ note_client.py      note 検索 API クライアント
│  ├─ analyzer.py         売れ筋・タイトル傾向の分析
│  ├─ reporter.py         CSV / Markdown レポート出力(AI考察含む)
│  └─ logger.py
└─ output/                結果の出力先
```
