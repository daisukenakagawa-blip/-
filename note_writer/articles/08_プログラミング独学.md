# 未経験から独学3ヶ月でポートフォリオを作る学習ロードマップ

「プログラミングを独学したいけど、何を・どの順で学べばいい?」
「教材が多すぎて、いつまでも入門書を彷徨ってしまう」——

このnoteは、**挫折せずに3ヶ月で"作品（ポートフォリオ）"を1つ完成させる**ことに
絞った学習ロードマップです。広く浅く学ぶのではなく、**作りながら覚える**順番を示します。

## このnoteで分かること

- 独学で挫折する3大原因と回避策
- 3ヶ月の週別カリキュラム
- 「写経 → 改造 → 自作」の進め方
- ポートフォリオに何を作ればいいか

## 独学で挫折する3大原因

1. **入門書を何冊も読む**だけで、手を動かさない
2. **環境構築でつまずいて**そのまま離脱
3. **作るものが決まっていない**ので学習が終わらない

対策はシンプル。「**最初に作るもの（ゴール）を1つ決める**」ことです。

## 1. 最初の言語の選び方

迷ったら、目的で決めます。

| 作りたいもの | おすすめ |
|-------------|---------|
| Webサイト/Webアプリ | HTML/CSS → JavaScript |
| 業務自動化・データ処理 | Python |
| スマホアプリ | Dart(Flutter) など |

> 「人気だから」ではなく「**自分が作りたいものに最短の言語**」を選びます。

## 2. 3ヶ月カリキュラム（週別）

**1ヶ月目：基礎の手を動かす**
- Week1：環境構築＋HTML/CSSで簡単なページ
- Week2：JavaScript（or Python）の基本文法を写経
- Week3：小さなプログラムを10個写経
- Week4：写経したコードを「少し改造」する

**2ヶ月目：作りながら覚える**
- 簡単なアプリ（ToDoリスト、電卓、メモ帳など）を自作
- 分からない箇所はAIに質問しながら進める

**3ヶ月目：ポートフォリオを仕上げる**
- 自分のテーマで1作品を完成（後述）
- GitHubに公開し、READMEを書く

## 3. 「写経 → 改造 → 自作」の黄金ステップ

1. **写経**：お手本を一字一句書き写す（仕組みを体に入れる）
2. **改造**：色・文言・機能を少し変える（理解を確認）
3. **自作**：ゼロから自分で作る（実力になる）

いきなり自作しようとして挫折する人が多い。**必ず写経から**始めます。

## 4. ポートフォリオは何を作る?

- ToDoアプリ（定番・基礎が詰まっている）
- 家計簿アプリ（データ保存を学べる）
- 自分の趣味に関するWebサイトやAPI

> 凝りすぎ厳禁。**「動く・公開してある・READMEがある」**の3点が揃えば十分です。

## 5. つまずいたときの調べ方

- エラー文をそのままコピーして検索 or AIに貼る
- 公式ドキュメントを「一次情報」として確認
- 30分悩んだら質問する（時間を溶かさない）

## まとめ

独学成功の鍵は「作るものを決める → 写経 → 改造 → 自作」。
まずは3ヶ月後に作るポートフォリオを1つ決めましょう。それが学習の地図になります。
